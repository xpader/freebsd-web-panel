# 16 — 定时任务（crontab）

## 概述

管理 cron 计划任务，覆盖两类来源并按来源分区展示：

- **`/etc/crontab`**（系统格式，6 字段，多一列 `who` 指定运行用户）—— 直接读写文件，cron 依 mtime ~1 分钟内自动重载。
- **`/var/cron/tabs/<用户>`**（用户格式，5 字段，运行用户=文件名）—— 读时直接扫目录读文件；写时通过 `crontab -u <用户> -` 管道提交，由 cron 校验语法并重新生成它自己管理的文件头。

前端把每个来源渲染成一个独立分区（一行标题表头），标题为 `/etc/crontab` 或用户名；每个分区右侧有自己的「+ 添加」按钮表示创建位置。每条任务支持编辑、删除、启用/禁用切换。任务上方紧邻的 `#` 注释行作为该任务的「注释」字段（可多行、可编辑）。

## 实现细节

### 后端 `src/handlers/crontab.rs`

#### 来源与文件头保护

两类来源各有「受保护前导（preamble）」，读取时不解析为条目，写入时原样回写：

- `/etc/crontab`：从文件开头到**列名图例行**（`#minute\thour\tmday\tmonth\twday\twho\tcommand`，制表符分隔）为止。用正则 `^#\s*minute\s+hour\s+mday\s+month\s+wday\s+who\s+command` 定位（`\s+` 兼容空格/制表符）。图例缺失时 preamble=0（此时头部注释/SHELL=/PATH= 仍作为 inert 块原样保留，只是会被算法视作普通注释，可能误挂到首条任务——标准文件总有图例行，故不影响）。
- 用户 tab：开头连续的自动生成行——`# DO NOT EDIT THIS FILE …`、`# (… installed on …)`、`# (Cron version …)`。按模式匹配跳过。**写入时不回写这部分**，因为 `crontab -u -` 会自动重新生成。

`preamble_len(lines, is_system)` 按来源类型返回前导行数。

#### 解析模型（块结构）

文件被建模为「preamble + 有序块列表」，保证无损往返：

```rust
enum Block {
    Inert(String),          // 空行 / 环境变量(NAME=VALUE) / 孤儿注释 —— 原样保留
    Entry(EntryBlock),      // 可编辑的计划任务
}
struct EntryBlock {
    comment_raw: Vec<String>,   // 该任务上方紧邻的注释原始行（含 #）
    task_idx: usize,            // 任务行在整文件中的绝对行号（API 句柄）
    raw_task: String,           // 原始任务行（未改动时原样回写）
    parsed: Parsed,             // 结构化字段
    disabled: bool,             // true = 整行被 # 注释（禁用任务）
}
```

`parse_body(rest, offset, is_system)` 逐行扫描：

- **空行**：中断注释归属；缓冲的注释 flush 为 inert。
- **`#` 开头**：去掉 `#` 与一个空格得 body：
  - body 经 `classify` 能解析为计划 → **禁用任务**（缓冲注释成为它的 comment）。
  - body 为空（单独 `#`）→ inert 分隔行。
  - 否则 → 加入注释缓冲。
- **非注释非空**：`classify` 能解析 → **启用任务**（缓冲注释成为它的 comment）；否则（环境变量/乱码）→ inert。

**注释归属规则**：紧邻某任务上方、连续的（中间无空行/非注释行）注释行都属于该任务；注释文本 = 每行去掉 `#` 与一个前导空格后用 `\n` 连接。

#### `classify` 的关键字段校验（避免散文误判）

`# Save some entropy so that /dev/random can re-seed on boot.` 这种散文注释有 ≥7 个空白词，会被朴素解析当成「禁用任务」。为区分散文与真正的 `#0 0 1 */2 * root certbot …` 禁用任务，`classify` 在计划分支强制校验**前 5 个时间字段**必须匹配 `^[0-9*/,\-]+$`（仅数字与 cron 元字符）。散文词（`Save`/`some`/`entropy`）含字母 → 不匹配 → 返回 `None` → 视为注释。真正的 cron 字段（`0`/`*/2`/`1-5`）通过。

#### 系统自带任务标识（`system_task`）

FreeBSD 默认 `/etc/crontab` 预装了一批基本系统维护任务。`is_freebsd_builtin(source, command)` 在 `entry_to_cron` 里为这些任务置 `system_task: true`，供前端用橙色 badge 标识（纯展示，不阻止编辑/删除）。识别依据是命令首词——这些是基本系统二进制，用户一般不会自行调度：

| 命令 | 任务 |
|---|---|
| `/usr/libexec/save-entropy` | 熵保存 |
| `newsyslog` | 日志轮转 |
| `periodic daily/weekly/monthly` | 定期维护 |
| `adjkerntz` | 时区校正 |

仅 `source == "system"` 时判定（用户 tab 里即便跑 `periodic` 也不算系统任务）。

> 副作用：不支持月/周名（`jan`/`mon`）。含字母的时间字段会被当作注释。实际 crontab 几乎都用数字，权衡可接受。

`@`-别名（`@reboot` 等）走单独分支，不参与时间字段校验。

#### 系统格式 vs 用户格式

`is_system` 决定字段数：系统多一个 `who` 列（计划 7 字段、`@`-别名 3 字段）；用户 5/2 字段。

##### 创建目标枚举（`targets`）

`GET /api/crontab/targets` 返回可选的创建目标，供主添加弹窗的下拉选择。覆盖 `/etc/crontab` 加所有「可能拥有 crontab」的用户，使得**尚无 crontab 的用户也能被选为目标**：

- `/etc/crontab`（source=`system`）
- 用户集 = `{root}` ∪ `/var/cron/tabs/` 中已有 tab 的用户 ∪ `/etc/passwd` 中 UID≥1000 的真实登录账户。BTreeSet 去重排序，用户名经 `RE_USERNAME` 校验（防异常字符）。

每项带 `has_tab`（是否已有 crontab 文件），前端可据此提示。

### 写入（按来源分流，原子）

`write_source(source, is_system, preamble, blocks)`：

- **系统**：`preamble 逐行 + serialize_body(blocks)` 拼成全文，`atomic_write` 写 `/etc/crontab`（写 `<path>.fwp.tmp` → chmod 0644 → rename，原子替换）。
- **用户**：`serialize_body(blocks)` 得到正文（**不含 preamble**），管道给 `crontab -u <name> -`。cron 校验语法后才安装，并自行重生成 3 行文件头。正文为空时改用 `crontab -u <name> -r`（删除整个 tab，等价于无任务）。

`serialize_body`：inert 块输出原行；entry 块输出各注释原始行 + 任务原始行；每行末尾加 `\n`。**未改动的 entry 用原始行回写（无损）；仅被编辑的 entry 用重建的注释行 + 重建任务行替换。**

#### 输入校验

- `validate_field`：5 个时间字段非空、≤128、禁 `\0`/`\n`/`\r`。
- `validate_command`：trim 后非空、≤4096、禁换行/空字节。
- `validate_special`：必须在已知 `@`-别名集合内。
- `validate_username`：`who` 列与 source 用户名须匹配 `^[a-zA-Z0-9_.-]{1,32}$`（防路径穿越，source 非 `system` 时拼 `/var/cron/tabs/<source>`）。
- `validate_comment`：≤2000、禁 `\0`（允许换行，按行拆分）。
- 用户 tab 写入还经 `crontab -` 二次语法校验；系统文件无此校验，靠上述字段检查 + 原样保留注释/环境变量兜底。

#### 重建任务行

`build_task(input, is_system)` → `(正文行, Parsed)`：
- `@`-别名非空：系统 `@sp user cmd`，用户 `@sp cmd`。
- 否则 5 字段：系统 `m h dom mon dow user cmd`，用户 `m h dom mon dow cmd`。
- `disabled` 为真时整体加 `# ` 前缀。
注释行 `build_comment_lines`：按 `\n` 拆分，过滤空行，每行加 `# ` 前缀。

### 数据结构（API）

```rust
struct CronEntry {
    source: String,          // "system" | 用户名
    line: usize,             // 任务行绝对行号（编辑/删除句柄）
    kind: String,            // "schedule" | "special"
    minute/hour/dom/month/dow: Option<String>,  // schedule 才有
    special: Option<String>,                     // special 才有
    user: Option<String>,    // who 列（仅系统格式）
    command: String,
    comment: String,         // 注释，\n 连接（可编辑）
    disabled: bool,
    system_task: bool,       // FreeBSD 自带系统任务（save-entropy/newsyslog/periodic/adjkerntz）
}

struct EntryInput { minute, hour, dom, month, dow, special, user, command, comment, disabled }  // 条目字段
struct CreateRequest { source: String, #[serde(flatten)] entry: EntryInput }
struct UpdateRequest { source: String, line: usize, #[serde(flatten)] entry: EntryInput }
struct SourceLineQuery { source: String, line: usize }
```

### 前端 `web/js/pages/cron.js`

- `renderCron` 渲染 6 列表格：计划 / 用户 / 命令 / 注释 / 状态 / 操作。
- `orderedSources`：system 在前，其余用户名字母序。每来源一个 `.cron-section-row`（跨 6 列），含标题（system→`/etc/crontab`，否则用户名）、条目计数、右侧「+ 添加」按钮（`__fwpCronAdd(source)`）。
- 全量加载后客户端筛选（计划/命令/用户/注释/来源）。
- 自定义弹窗 `entryModal(title, preselect, entry, submitLabel)`：
  - 顶部「目标」下拉（`/api/crontab/targets` 拉取，缓存）：`/etc/crontab` + 真实系统用户（已有 tab 的 + UID≥1000 登录账户 + root）。让主添加按钮能为**尚无 crontab 的用户**创建任务。`preselect` 决定初始选中项；编辑时该下拉 `disabled`（不支持跨文件移动任务）。
  - 目标切换实时控制运行用户（`who`）字段的显隐：仅 `/etc/crontab` 显示（系统格式 6 字段），用户 tab 隐藏（运行用户=目标用户名）。
  - 「计划类型」下拉（自定义 / `@`-别名），选自定义时显示 5 列时间字段栅格。
  - 命令（必填）、注释 textarea（多行）、「禁用」复选框。
- 两处添加入口共用同一弹窗：
  - 工具栏右侧主「+ 添加任务」按钮（`__fwpCronAdd()` 无参）→ 目标可选，默认 system。
  - 每个分区表头右侧「+ 添加」按钮（`__fwpCronAdd(source)`）→ 预选该来源，仍可改。
- 全局句柄：`__fwpCronAdd/Edit/Toggle/Del`，均带 `source`；每次操作后 `load()` 刷新（行号随之更新）。
- `@`-别名附带可读描述（`cron.alias_*`）。禁用行 `.row-dim` 半透明。HTML 经 `esc()`、属性经 `escAttr()` 转义。

## API

| 方法 | 路径 | 请求 | 响应 |

## 已知限制 / TODO

- 仅管理 `/etc/crontab` 与 `/var/cron/tabs/*`；**不**管理 `/etc/cron.d/`、`/usr/local/etc/cron.d/`（按需求范围排除）。
- 删除某用户 tab 的最后一条 → 整个 tab 被 `crontab -r` 移除（等价于无任务；该来源下次不再出现在分区列表，除非重新创建）。
- 不单独展示/编辑环境变量（SHELL/PATH/MAILTO）与孤儿注释——原样保留在文件中，不在列表暴露。
- 月/周日名（`jan`/`mon`）不被识别为时间字段（含字母），含此类字段的行会被当作注释。实际 crontab 几乎都用数字。
- 含字母的注释若恰好形如 cron 计划（`#0 0 * * * root x`），会被识别为禁用任务而非注释——这是 cron 格式固有的二义性（注释掉的命令与"长得像命令的注释"无法区分）。
- 写系统文件为直接改文件（原子 tmp+rename），无 `crontab` 语法校验，靠字段校验 + 原样保留兜底；用户 tab 写入有 `crontab -` 校验。
- 行号为读时快照；并发改动存在理论竞争窗口（单管理员场景可忽略），每次操作后重新拉取以缩小窗口。
- 运行以 root 身份，可读写所有用户 tab；无额外权限分级。
