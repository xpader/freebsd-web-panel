# 12 — 国际化（i18n）

## 概述

前端多语言界面，基于成熟 i18n 框架 **i18next**（UMD 构建，vendored 到 `web/vendor/`）实现。当前支持简体中文（`zh`，项目默认）与英文（`en`）。语言偏好持久化在 `localStorage`，顶栏右侧（设置按钮左侧）用国旗图标下拉切换，切换后当前页面立即重渲染。

技术约束与项目一致：纯 ES Module、无构建步骤。i18next 以 UMD 全局变量方式懒加载（与 Chart.js 同模式），不引入 npm/bundler。

## 实现细节

### 库与资源

- **i18next UMD**：`web/vendor/i18next.min.js`（v23.x，约 48 KB），由 `rust-embed` 内嵌、`web_assets::serve_asset` 托管于 `/vendor/i18next.min.js`
- **资源**：`web/js/i18n/translations.js` 导出 `en` / `zh` 两个对象，键按领域分组（`common`、`nav`、`topbar`、`auth`、`dash`、`fs`、`zfs` 等），值用 i18next 插值语法 `{{name}}`

`common` 命名空间存放**跨域复用**的通用词条，遵循以下归类规则：
- UI 动作/通用列名：`cancel`、`confirm`、`delete`、`actions`、`name`、`size`、`type`、`device`…
- 详情页导航返回：`navBack`（`← 返回` / `← Back`）
- 存储术语（被 fs/disks/zfs 共用）：`used`、`free`、`capacity`、`frag`、`dedup`
- 通用表头：`moduleStatus`、`colCreatedAt`（创建时间）
- 含义宽泛的通用词（如 `network`「网络」）只放 `common`，不在 `nav` 等场景空间重复定义

### 核心原则：同一含义只用一个 key

同一个词（如「网络」「所有者」「权限」）出现在菜单、列表、弹窗、编辑器等不同位置时，必须共用同一个 key，不要按使用场景拆成多个。

**反面教材**：`nav.network`（顶级菜单「网络」）+ `nav.monitorNetwork`（监控侧边栏「网络」）— 两个 key，翻译完全相同，只因位置不同而拆分。正确做法：`common.network`，导航/监控侧边栏等所有位置复用。

`nav` 等场景空间只放该场景**特有短语**（如 `nav.networkIf`「网络接口」），与 `common` 同义的词一律复用，不新建。

`state`（设备健康状态，如 ONLINE）与 `status`（任务/模块状态，如 Planned）**语义不同，分域保留**，不合并入 `common`。

### 包装模块 `web/js/i18n/index.js`

对 i18next 的薄封装，对外暴露同步 API：

| 导出 | 说明 |
|---|---|
| `initI18n()` | 懒加载 UMD 脚本（`<script>` 注入），`i18next.init({ lng, fallbackLng:'en', resources, returnEmptyString:false })`。幂等，返回缓存 Promise |
| `t(key, vars?)` | 同步翻译；i18next 未就绪时直接返回 `key` |
| `getLang()` | 当前语言码（`en`/`zh`） |
| `setLang(code)` | `i18next.changeLanguage` + 写入 `localStorage['fwp_lang']` + 派发 `window` 事件 `fwp:langchange` |
| `getLocale()` | `'zh-CN'` / `'en-US'`，供 `Date.toLocaleString()` 使用 |
| `LANGUAGES` | `[{code, label, flag}]`，国旗 emoji 作菜单图标 |
| `currentLangMeta()` | 当前语言的 `{code,label,flag}` 元数据 |

### 初始化时序 `web/js/main.js`

```js
initI18n().then(() => {
  window.addEventListener('fwp:langchange', reload);  // 切换语言后重渲染
  startRouter();
});
```

**关键约束**：`initI18n()` 必须在 `startRouter()` 之前 resolve，保证首次渲染时 `t()` 已可用。所有页面的 `t()` 调用因此可保持同步。

### 语言检测

`detectInitialLang()`：
1. 读 `localStorage['fwp_lang']`，合法则用之
2. 否则嗅探 `navigator.language`：`en*` → `en`，其余 → `zh`（项目中文优先）

### 切换菜单 `web/js/ui/layout.js`

顶栏右侧布局顺序（从左到右）：**语言切换 → 设置 → 用户**。语言切换放在设置左侧。

```
<div class="topbar-right">
  <div class="settings-menu" id="lang-menu">      <!-- 语言切换 -->
    <button class="lang-btn" id="lang-toggle">     <!-- 国旗图标按钮 -->
      <span class="lang-flag">🇨🇳</span>
    </button>
    <div class="settings-dropdown">               <!-- 复用 settings-dropdown 样式 -->
      <a class="lang-item" data-lang="en">🇺🇸 English</a>
      <a class="lang-item" data-lang="zh">🇨🇳 简体中文</a>
    </div>
  </div>
  <div class="settings-menu" id="settings-menu"> ... </div>
  <div class="settings-menu" id="user-menu"> ... </div>
</div>
```

- 触发按钮显示当前语言的国旗（`currentLangMeta().flag`）
- 点击下拉项调用 `setLang(code)` → 派发 `fwp:langchange` → `router.reload()` 重渲染当前路由
- `bindSettingsDocClick()` 已扩展为点击外部同时关闭语言菜单

### 菜单/标签的 i18n 化

`layout.js` 的 `MENU` / `SETTINGS` 数组中每项的字段从 `label` 改为 **`labelKey`**（i18n 键，如 `nav.dashboard`），在 `renderLayout` / `renderSidebarItem` 渲染时用 `t(item.labelKey)` 解析。这样切换语言无需改数据结构，只需重渲染。

### 日期本地化

`toLocaleString('zh-CN')` 的硬编码 locale 全部改为 `getLocale()`：
- `web/js/pages/users.js`、`audit.js`、`zfs.js` 的 `fmtTime`

## 外部依赖

- **i18next 23.x**（UMD，vendored，非 npm）——JS 生态最成熟的 i18n 框架
- 无其它新增依赖

## 配置项

无 `fwp.toml` 配置项。语言偏好纯前端存储（`localStorage['fwp_lang']`）。

## 已知限制 / TODO

- 仅 `en` / `zh` 两种语言；新增语言只需扩展 `translations.js` 与 `LANGUAGES`
- 后端消息（如校验错误）仍为固定字符串，未做 i18n；前端按 `err.message` 原样显示
- 数字/字节格式化（`fmtBytes`）未本地化（统一用 SI 简写 B/KB/MB…）
- 国旗用 emoji，个别平台渲染可能不同（🇺🇸 代表英文、🇨🇳 代表中文）
