# 设计：前端

## 1. 技术选型

**原生 SPA，无构建步骤。**

- HTML 入口 `index.html` + ES Modules（`<script type="module">`）
- 原生 CSS（自写深色主题）
- 路由：轻量 hash router（`#/jails`、`#/vms`），约 50 行自写
- 状态/请求：原生 `fetch` 封装，token 存 `sessionStorage`
- 组件：用 ES Module 导出返回 DOM 节点的工厂函数，无虚拟 DOM

> 不引入 React/Vue/构建工具。系统管理面板交互复杂度中等，原生 JS 足够，且消除构建链依赖、便于单机部署。后期若 UI 膨胀，可在不改后端 API 的前提下迁移。

## 2. 信息架构（顶部主菜单 + 左侧子菜单）

界面采用**两级导航**：顶部水平主菜单（一级），左侧垂直子菜单（二级，随主菜单切换）。

```
┌─────────────────────────────────────────────────────────────┐
│  FreeBSD Web Panel   [概览] [配置] [网络] [文件系统] [虚拟化] [系统]   user ▾ │
├──────────────┬──────────────────────────────────────────────┤
│  仪表盘       │                                              │
│              │                                              │
│  (子菜单)     │              主内容区                         │
│              │                                              │
│              │                                              │
│              │                                              │
│  ──────      │                                              │
│  退出登录     │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 主菜单与子菜单映射

| 顶部主菜单 | 左侧子菜单 | 对应路由 |
|---|---|---|
| **概览** | 仪表盘 | `/dashboard` |
| **配置** | Sysctl 系统参数 | `/sysctl` |
|  | RC 配置 | `/rcconf` |
|  | 服务管理 | `/services` |
| **网络** | 网络接口 | `/network` |
|  | 防火墙 (PF) | `/pf` |
| **文件系统** | ZFS | `/zfs` |
| **虚拟化** | Jail 容器 | `/jails` |
|  | Bhyve 虚拟机 | `/bhyve` |
| **系统** | 用户管理 | `/users` |
|  | 审计日志 | `/audit` |

### 导航规则
- 顶部主菜单点击 → 切换左侧子菜单组 + 导航到该组的默认页
- 左侧子菜单点击 → 导航到具体页面
- 当前页面所属的主菜单自动高亮；当前页面的子菜单项高亮
- 登录/初始化页不显示导航骨架

## 4. 页面规划

### 4.1 Dashboard（总览）
- 系统信息：hostname、OS 版本、uptime、负载
- 资源：CPU 使用率、内存、swap、磁盘（ZFS pool 健康）
- 资源计数：jail 数（运行/停止）、VM 数、最近告警

### 4.2 各功能页
每个模块一个主页面，列表为主，点击进入详情/编辑抽屉（drawer）或模态框。
危险操作（destroy/stop/rollback）用确认对话框，可选输入名称匹配才放行。

### 4.3 控制台
jail `jexec` 控制台和 VM `vm console` 用 xterm.js（ESM 本地副本）通过 WebSocket 连接后端 PTY 通道。

## 5. 实时数据
- Dashboard 资源指标：WebSocket 推送或轮询（每 2s）
- Jail/VM 状态：列表页轮询（每 5s）
- 控制台：长连接 WebSocket

## 6. 嵌入策略
- 开发期：磁盘 `web/` 优先（改文件即时生效）
- 生产期：`rust-embed` 将 `web/` 编译进二进制（`embed-web` 默认开启），任意目录运行可用

## 7. 实现里程碑

1. **M1 — 顶部+左侧两级导航骨架 + 路由 + API 封装 + 登录页**
2. **M2 — Dashboard + 通用表格/表单/对话框组件**
3. **M3 — sysctl/rc.conf/网络/服务页**
4. **M4 — Jail 列表/编辑页**
5. **M5 — VM 列表/详情页**
6. **M6 — ZFS 树形视图**
7. **M7 — 终端控制台（xterm + WebSocket）**
